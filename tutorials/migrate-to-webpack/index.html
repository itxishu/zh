A “normal” app

Let’s begin with a simple web app and let’s try to make use of webpack step by step.

add app.coffee

$ ->
    # Model:
    userAgent = window.navigator.userAgent

    # View:
    view =
        render: (ua) ->
            $("#user-agent").text ua

    # Controller (render the model):
    view.render userAgent

add index.html

<!DOCTYPE html>
<html>
    <head>
        <script type="text/javascript" charset="utf-8" src="jquery.min.js"></script>
        <script type="text/javascript" charset="utf-8" src="app.coffee.js"></script>
        <link rel="stylesheet" href="styles.less.css" />
    </head>
    <body>
        <h1>Hello World.</h1>
        <p>Your User-Agent is: <span id="user-agent"></span></p>
    </body>
</html>

add styles.less

body {
    background: #eee;
    p {
        border: 1px solid #333;
        border-radius: 3px;
    }
}

Publishing

For publishing this app, you first need to compile app.coffee to app.coffee.js and style.less to style.less.css. Than you may open the app.

Summary

That’s a really simple app.

Let’s webpack

In the first step we want to use webpack for bundling the single file.

We care for handling coffeescript in the next step. For now we just bundle the generated js file.

It’s assumed you have webpack installed. See others guides for instructions.

Create The Bundle

webpack app.coffee.js bundle.js

Use The Bundle

We can remove the app.coffee.js file from the html and use the bundled version instead:

update index.html

  <!DOCTYPE html>
  <html>
  	<head>
  		<script type="text/javascript" charset="utf-8" src="jquery.min.js"></script>
- 		<script type="text/javascript" charset="utf-8" src="app.coffee.js"></script>
+ 		<script type="text/javascript" charset="utf-8" src="bundle.js"></script>
  		<link rel="stylesheet" href="styles.less.css" />
  	</head>
  	<body>
  		<h1>Hello World.</h1>
  		<p>Your User-Agent is: <span id="user-agent"></span></p>
  	</body>
  </html>

Summary

The single file app.coffee.js now goes though webpack.

Preprocess coffeescript

We want webpack to compile coffeescript to javascript while compiling.

Install the coffeescript loader

npm install coffee-loader

Create The Bundle

webpack app.coffee bundle.js --module-bind coffee

This binds the extension .coffee to the loader coffee-loader.

Note: The generated file app.coffee.js can be deleted.

Summary

webpack now compiles coffeescript to javascript for us.

Use a configuration file

We want to put our configuration into a file to we don’t have to type all the command line arguments to compile.

webpack use webpack.config.js as configuration if found in the current directory

add webpack.config.js

module.exports = {
    // the entry point
    entry: "./app.coffee",

    output: {
        // the output path
        path: __dirname,

        // the output filename
        filename: "bundle.js"
    },

    module: {
        loaders: [
            // support for coffeescript files
            { test: /\.coffee$/, loader: "coffee-loader" }
        ]
    }
};

Create The Bundle

Now we just have to call webpack without arguments.

webpack

Summary

Our configuration is now in a file.

Let’s make modules

Because we use webpack now, we can use require to load other modules.

We first want to split our code into model, view and controller.

We use CommonJS style modules in this example, but AMD style modules are also possible (or mix both styles).

add userAgentModel.coffee

module.exports = window.navigator.userAgent

add userAgentView.coffee

exports.render = (model) ->
  $("#user-agent").text model

update app.coffee

  $ ->
- 	# Model:
- 	userAgent = window.navigator.userAgent
- 
- 	# View:
- 	view =
- 		render: (ua) ->
- 			$("#user-agent").text ua
- 
- 	# Controller (render the model):
- 	view.render userAgent
+ 	model = require "./userAgentModel.coffee"
+ 	view = require "./userAgentView.coffee"
+ 	view.render model

Summary

Webpack now takes care of our coffeescript files and bundle them together.

We use jQuery in the app. But wait jQuery is javascript. Should it not be a dependency which can be required?

Yes. Let’s create a jquery web module so it can be required:

jquery web module

  • Create a folder web_modules.
  • Put the jquery library into this folder as jquery.js

update app.coffee

+ $ = require("jquery")
  $ ->
  	model = require "./userAgentModel.coffee"
  	view = require "./userAgentView.coffee"
  	view.render model

update index.html

  <!DOCTYPE html>
  <html>
  	<head>
- 		<script type="text/javascript" charset="utf-8" src="jquery.min.js"></script>
  		<script type="text/javascript" charset="utf-8" src="bundle.js"></script>
  		<link rel="stylesheet" href="styles.less.css" />
  	</head>
  	<body>
  		<h1>Hello World.</h1>
  		<p>Your User-Agent is: <span id="user-agent"></span></p>
  	</body>
  </html>

update userAgentView.coffee

+ $ = require("jquery")
  exports.render = (model) ->
    $("#user-agent").text model

Not relative require calls are looked up in modules folders. For web modules we can use a folder "web_modules".

Summary

All javascript stuff is now bundled.

Our html still contains another static resource: style.less.css (compiled from style.less).

We want it bundled too! With webpack that’s easy:

stylesheets

We need more loaders:

  • One to transform less to css: less-loader
  • One to transform css to javascript: css-loader
  • One to add css to the DOM: style-loader
npm install less-loader css-loader style-loader

These loaders need to be applied for less files. We can configure this in the configuration and just require the stylesheet.

update index.html

  <!DOCTYPE html>
  <html>
  	<head>
  		<script type="text/javascript" charset="utf-8" src="bundle.js"></script>
- 		<link rel="stylesheet" href="styles.less.css" />
  	</head>
  	<body>
  		<h1>Hello World.</h1>
  		<p>Your User-Agent is: <span id="user-agent"></span></p>
  	</body>
  </html>

update userAgentView.coffee

+ require "./styles.less"
  $ = require("jquery")
  exports.render = (model) ->
    $("#user-agent").text model

update webpack.config.js

  module.exports = {
  	// the entry point
  	entry: "./app.coffee",
  
  	output: {
  		// the output path
  		path: __dirname,
  
  		// the output filename
  		filename: "bundle.js"
  	},
  
  	module: {
  		loaders: [
+ 			// support for less files
+ 			{ test: /\.less$/, loader: "style-loader!css-loader!less-loader" },
+ 
  			// support for coffeescript files
  			{ test: /\.coffee$/, loader: "coffee-loader" }
  		]
  	}
  };

Because webpack can handle less the extra step of compiling the less file can be omitted.

Note: You can remove the generated file styles.less.css.

Summary

We moved all referenced static resources into the bundle…

Note: If you write a single page application this step is useful, but if you write a progressively enhanced application you shouldn’t do it.

Our html still contains another static resource: style.less.css (compiled from style.less).

… but there is still a static resource hidden. The html code.

You guess it, we can bundle it too. Because I like jade templates:

npm install jade-loader

add userAgentView.jade

h1 Hello World.
p Your User-Agent is: #{model}

update index.html

  <!DOCTYPE html>
  <html>
  	<head>
  		<script type="text/javascript" charset="utf-8" src="bundle.js"></script>
  	</head>
  	<body>
- 		<h1>Hello World.</h1>
- 		<p>Your User-Agent is: <span id="user-agent"></span></p>
  	</body>
  </html>

update userAgentView.coffee

  require "./styles.less"
  $ = require("jquery")
+ template = require "./userAgentView.jade"
  exports.render = (model) ->
-   $("#user-agent").text model
+   $("body").html template
+     model: model

update webpack.config.js

  module.exports = {
  	// the entry point
  	entry: "./app.coffee",
  
  	output: {
  		// the output path
  		path: __dirname,
  
  		// the output filename
  		filename: "bundle.js"
  	},
  
  	module: {
  		loaders: [
+ 			// support for jade files
+ 			{ test: /\.jade$/, loader: "jade-loader" },
+ 
  			// support for less files
  			{ test: /\.less$/, loader: "style-loader!css-loader!less-loader" },
  
  			// support for coffeescript files
  			{ test: /\.coffee$/, loader: "coffee-loader" }
  		]
  	}
  };

Summary

So finally we really moved all static resources into the bundle!

Note: This is again of course not suitable for a progressively enhanced application.
